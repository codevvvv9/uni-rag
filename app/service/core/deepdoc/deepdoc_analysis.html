DeepDOC 来源与目录拆解

分析 backend/app/service/core/deepdoc 的来源证据、目录内容、模型依赖和面试讲法。

1. 来源结论

结论：该目录不是本项目纯手写模块，而是基于 InfiniFlow/RAGFlow 的 DeepDoc 代码迁移并适配到 service.core。
  • 源码文件头保留 InfiniFlow Authors 和 Apache-2.0 许可。
  • README 图片资源指向 github.com/infiniflow/ragflow。
  • 类名保留 RAGFlow*Parser 命名。
  • 模型仓库使用 InfiniFlow/deepdoc 与 InfiniFlow/text_concat_xgb_v1.0。
  • 仓库历史中存在旧路径 app/service/ragflow/deepdoc。

参考：RAGFlow DeepDoc README，RAGFlow GitHub，deepdoc_pdfparser PyPI。

Parser

  • pdf_parser.py：OCR、版面、表格、坐标。
  • docx_parser.py：段落、样式和表格。
  • excel_parser.py：行列转文本或 HTML。
  • json_parser.py：保持层级的 JSON 切块。

Vision

  • ocr.py：文本检测和识别。
  • layout_recognizer.py：版面区域识别。
  • table_structure_recognizer.py：表格结构识别。
  • recognizer.py：排序、重叠匹配和清理。

4. 模型资源依赖

deepdoc 代码依赖 backend/app/service/core/rag/res/deepdoc 中的 ONNX 和 XGBoost 模型；本地缺失时会尝试从 InfiniFlow 的 HuggingFace 仓库下载。

flowchart LR
  A["ocr.py"] --> B["det.onnx / rec.onnx / ocr.res"]
  C["layout_recognizer.py"] --> D["layout*.onnx"]
  E["table_structure_recognizer.py"] --> F["tsr.onnx"]
  G["pdf_parser.py"] --> H["updown_concat_xgb.model"]
  I["缺少本地模型"] --> J["InfiniFlow/deepdoc"]
  I --> K["InfiniFlow/text_concat_xgb_v1.0"]

5. 适配痕迹与面试讲法

适配痕迹

  • 主链路 import 已改为 service.core.*。
  • 模型路径绑定到 rag/res/deepdoc。
  • 部分 resume 和测试脚本仍保留上游式 import。

面试讲法

  • 不是从零手写，是引入并适配 RAGFlow DeepDoc。
  • 工程重点是接入统一解析、分块、向量化和 ES 入库链路。
  • 核心价值是 OCR、版面识别和表格结构识别。
  • 亮点是保留表格、图片、坐标，支持 RAG 引用定位。
